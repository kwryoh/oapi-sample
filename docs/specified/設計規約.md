# 設計規約

## バージョン

- OpenAPI v3.0

※oapi-codegenの対応バージョン

## paths

### tags

- 必須
- 1URIで1つのタグのみ定義する
- リソース名を単数形で記載する
- キャメルケース

```yaml
# good
postid: ""
tag:
  - product # GoアプリケーションのhandlerやTypeScriptのclassの単位となる

# bad
postid: ""
tag:
  - products

# bad
postid: ""
tag:
  - user
  - product
```

### operationId

- 必須
- `${HTTPメソッド}${機能物理名}`を記載する
- キャメルケース

```yaml
# GET /products
operationId: getProducts
# GET /products/:product_id
operationId: getProductByProductId
# POST /products
operationId: postProducts
# PUT /products/:product_id
operationId: putProduct
# DELETE /products/:product_id
operationId: DeleteProduct
```

### summary

- 必須
- ${機能論理名} で定義する

```yaml
summary: 商品参照
```

### security

- 必須
- 認証の要否で以下のように定義する

```yaml
# 認証なし
security: []

# 認証あり
security:
  - isAuthorized: []

# OAuth認証
securityDefinitions:
  isAUthorized:
    type: 
```
